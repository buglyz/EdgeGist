# EdgeGist WebUI 重构方案 - 论文与代码版本控制优化

## 🎯 用户需求分析

### 主要使用场景
1. **学术论文管理**
   - LaTeX 文件版本控制
   - Markdown 论文写作
   - 参考文献管理 (.bib)
   - 多人协作草稿

2. **代码版本控制**
   - 代码片段收藏
   - 多版本对比
   - 变更历史追踪
   - 快速回退和恢复

### 当前 UI 痛点
1. 文件列表缺少分类和标签
2. 版本历史不够直观
3. 缺少 Markdown/LaTeX 实时预览
4. 搜索功能不够强大（无法按类型/标签过滤）
5. 没有针对学术写作的特性（字数统计、引用管理）
6. 代码 diff 可以更智能

## 🛠️ 技术实现方案

### 阶段 1: 核心功能增强 (优先)
1. **标签系统**
   - 数据库：新增 `gist_tags` 表
   - API：`POST /gists/:id/tags`, `GET /gists/tags`
   - UI：标签选择器 + 过滤

2. **统计信息**
   - 字数统计（中英文分别计算）
   - 行数统计
   - 文件类型分析

3. **Markdown 实时预览**
   - 库：`marked` + `KaTeX`
   - 编辑器：CodeMirror 6 或保持现有
   - 分屏显示

### 阶段 2: 高级特性

   - 引用跳转

5. **智能搜索**
   - D1 全文索引

   - 搜索历史

6. **版本时间轴**
   - 可视化组件
   - 快速回退
   - 变更统计

7. **评论系统**

## 🎨 设计原则

1. **简洁优先** - 不过度设计，保持界面清爽
2. **性能为先** - 大文件优化，懒加载

4. **响应式** - 移动端友好
5. **可访问性** - 键盘导航，屏幕阅读器支持
